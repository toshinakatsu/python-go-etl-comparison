# PythonとGoを使ったETLの比較
## このリポジトリについて

本リポジトリでは、まず Python で最小構成の ETL を実装し、その後に同じ処理を Go で実装していき、
それぞれの特徴を比較・理解したり、用途やスケールに応じた選択ができるようになること
をこのリポジトリのゴール・目的としています。

- なぜデータ処理では Python がよく使われるのか
- Go で同じことをやると、何がシンプルになり、何が増えるのか
- 小規模 ETL から中規模・業務利用に進むとき、どちらが扱いやすいのか

- 型の有無による実装の違い
- エラーハンドリングの考え方
- 実行ファイルとして配布しやすい点
- 中長期で保守するコードの書き心地

といった視点で、実装と README の両方に学びを残していきます。

※ 実装はいずれも「学習用としての最小構成」を重視しています。
※ あえてフレームワークや外部サービスは使わず、標準ライブラリ中心で実装していきます。
